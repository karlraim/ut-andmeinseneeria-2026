# Praktikum 5: Suurandmed ja pilvelahendused (Baastase)

## Eesmärk

Tutvuda suurandmete põhimõtete ja pilve kasutusloogikaga. Praktikumi lõpuks mõistad, miks kasutatakse Parquet formaati, mis on hajussüsteem ja oskad teha esmaseid liigutusi Databricks keskkonnas.

## Õpiväljundid

Praktikumi lõpuks osaleja:

- Mõistab, miks kasutatakse Parquet formaati ja avatud tabeliformaate
- Saab aru, mida tähendab hajussüsteem ja miks seda vaja on
- Oskab teha esmaseid liigutusi Databricks keskkonnas
- Tunneb erinevust lokaalse ja pilvepõhise andmetöötluse vahel

## Teemad

| Teema | Kirjeldus |
|-------|-----------|
| Parquet | Veerupõhine andmeformaat ja selle eelised |
| Avatud tabeliformaadid | Delta Lake, Apache Iceberg, Apache Hudi |
| Hajussüsteemid | Miks ja kuidas andmeid hajutatakse |
| Databricks | Pilvepõhine andmeplatvorm ja selle kasutamine |

## Eeldused

- Eelmiste praktikumide kogemus andmete laadimise ja töötlemisega
- Põhiteadmised SQL-ist ja Python-ist

## Uued mõisted

| Mõiste | Selgitus |
|--------|----------|
| **Parquet** | Veerupõhine binaarne failivorming, optimeeritud analüütiliste päringute jaoks |
| **Data Lake** | Suur struktureerimata andmete hoidla, mis toetab erinevaid formaate |
| **Data Lakehouse** | Data Lake + Data Warehouse omaduste kombinatsioon |
| **Hajussüsteem** | Süsteem, kus andmeid ja arvutust jagatakse mitme masina vahel |
| **Databricks** | Pilvepõhine platvorm andmete töötlemiseks, analüüsiks ja masinõppeks |
