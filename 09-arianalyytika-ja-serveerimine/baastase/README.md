# Praktikum 9: Ärianalüütika ja andmete serveerimine (Baastase)

## Eesmärk

Seome tehnilise töö äriväärtusega. Praktikumi lõpuks suudad luua lihtsa näidikulaua (dashboardi), selgitada KPI-sid ja kirjeldada, mida andmetest saab järeldada.

## Õpiväljundid

Praktikumi lõpuks osaleja:

- Suudab luua lihtsa näidikulaua (dashboardi)
- Oskab selgitada 2–3 KPI-d ja nende tähendust
- Oskab kirja panna, mida andmetest saab järeldada ja mida mitte
- Mõistab andmepõhise loo kujundamise põhimõtteid

## Teemad

| Teema | Kirjeldus |
|-------|-----------|
| KPI-d | Mis on KPI ja kuidas neid defineerida |
| Dashboard | Näidikulaua loomine ja disainipõhimõtted |
| Andmelugu | Andmetest järelduste tegemine ja esitlemine |
| Piirangud | Mida andmed ei ütle — piirangute teadvustamine |

## Eeldused

- Eelmiste praktikumide kogemus andmete laadimise ja töötlemisega
- Põhiteadmised SQL-ist (agregatsioonid, GROUP BY)
- Töötav andmebaas eelmistest praktikumidest

## Uued mõisted

| Mõiste | Selgitus |
|--------|----------|
| **KPI** | Key Performance Indicator — võtmenäitaja, mis mõõdab äriliselt olulist tulemust |
| **Dashboard** | Visuaalne koondvaade, mis näitab olulisemaid mõõdikuid ja trende |
| **Andmelugu** | Andmete esitlemine loogilise narratiivina, mis toetab otsustamist |
| **Serveerimine** | Andmete kättesaadavaks tegemine lõppkasutajatele (API, dashboard, raport) |
