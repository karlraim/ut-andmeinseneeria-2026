# Praktikum 6: Andmekvaliteet ja andmehaldus (Edasijõudnud)

## Eesmärk

Automatiseerida andmekvaliteedi testid ja luua kvaliteedi raport. Praktikumi lõpuks mõistad andmete pärinevust, andmekataloogi rolli ja oskad kirjeldada andmehaldusega seotud rolle.

## Õpiväljundid

Praktikumi lõpuks osaleja:

- Suudab automatiseerida andmekvaliteedi testid töövoos
- Oskab luua andmekvaliteedi raporti
- Mõistab andmete pärinevuse (data lineage) kontseptsiooni
- Tunneb andmekataloogi rolli ja oskab kirjeldada andmehaldusega seotud rolle

## Teemad

| Teema | Kirjeldus |
|-------|-----------|
| Automatiseeritud testid | Kvaliteedikontrollide sidumine andmetöövoogu |
| Kvaliteediraport | Andmekvaliteedi koondülevaate genereerimine |
| Data Lineage | Andmete pärinevuse ja liikumise jälgimine |
| Andmekataloog | OpenMetadata vms tööriist andmevarade dokumenteerimiseks |
| Andmehalduse rollid | Data Owner, Data Steward, Data Engineer rollide eristamine |

## Eeldused

- Docker ja Docker Compose on paigaldatud
- Kogemus PostgreSQL, SQL ja Python-iga
- Arusaam andmekvaliteedi reeglitest (baastase teadmised)
- Töötav andmetorustik eelmistest praktikumidest

## Uued mõisted

| Mõiste | Selgitus |
|--------|----------|
| **Data Lineage** | Andmete pärinevus — visuaalne ülevaade, kust andmed tulevad ja kuhu lähevad |
| **Data Owner** | Äripool, kes vastutab andmekogu eest ja määrab ligipääsureeglid |
| **Data Steward** | Isik, kes hoolitseb andmekvaliteedi ja dokumentatsiooni eest |
| **Data Contract** | Kokkulepe andmete struktuuri ja kvaliteedi kohta tootja ja tarbija vahel |
| **OpenMetadata** | Avatud lähtekoodiga andmekataloogi ja metaandmete halduse platvorm |
