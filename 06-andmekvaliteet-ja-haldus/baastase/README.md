# Praktikum 6: Andmekvaliteet ja andmehaldus (Baastase)

## Eesmärk

Õppida rakendama kvaliteedikontrolle andmetöövoogudes. Praktikumi lõpuks suudad lisada kvaliteedireegleid, tuvastada vigaseid ridu ja mõistad andmekataloogide väärtust.

## Õpiväljundid

Praktikumi lõpuks osaleja:

- Suudab lisada vähemalt 5 kvaliteedireeglit andmetöövoogu
- Oskab näidata, kuidas vigased read tuvastatakse
- Mõistab, miks kasutatakse andmekatalooge
- Teab, mis väärtus on andmevarade kirjeldustel ja metaandmetel

## Teemad

| Teema | Kirjeldus |
|-------|-----------|
| Kvaliteedireeglid | NOT NULL, unikaalsus, vahemiku kontrollid, formaadi kontrollid |
| Vigade tuvastamine | Vigaste ridade leidmine ja raporteerimine |
| Andmekataloog | Andmevarade kirjeldamine ja dokumenteerimine |
| Metaandmed | Andmete kohta käivad andmed ja nende haldamine |

## Eeldused

- Docker ja Docker Compose on paigaldatud
- Eelmiste praktikumide kogemus PostgreSQL, SQL ja Python-iga
- Töötav andmetorustik eelmistest praktikumidest

## Uued mõisted

| Mõiste | Selgitus |
|--------|----------|
| **Andmekvaliteet** | Andmete täpsus, täielikkus, konsistentsus ja ajakohasus |
| **Kvaliteedireegel** | Kontroll, mis valideerib andmete vastavust ootustele |
| **Andmekataloog** | Tsentraalne register, mis kirjeldab organisatsiooni andmevarasid |
| **Metaandmed** | Andmete kohta käivad kirjeldavad andmed (skeem, päritolu, omanik) |
| **Data Lineage** | Andmete pärinevus — kust andmed tulevad ja kuhu lähevad |
